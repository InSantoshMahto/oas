# Contributing

Thanks for your interest in contributing to this project! We welcome contributions of all kinds — code, docs, bug reports, and design feedback.

How to contribute

1. Open an issue to discuss significant changes before you start work.
2. Fork the repository and create a feature branch: `git checkout -b my-feature`.
3. Make your changes in the `src/` directory. Keep changes focused and well-documented.
4. Run the build locally and include tests where appropriate:
   - `cd src`
   - `npm install`
   - `npm run build`

Pull request checklist

- [ ] The PR has a clear description of the change and why it's needed.
- [ ] Related issues are linked (if applicable).
- [ ] All public APIs are documented or have examples.
- [ ] CI passes (if present).

Coding guidelines

- Follow existing code style in the repository.
- Write small, focused commits with descriptive messages.
- Add unit tests for new behaviour where practical.

Reporting security issues

If you've found a security vulnerability, please consult `SECURITY.md` for instructions on responsible disclosure.

Code of conduct

This project follows the Code of Conduct in `CODE_OF_CONDUCT.md`. Please read it and be kind and respectful to other contributors.
